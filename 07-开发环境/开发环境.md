## 请介绍一下 git

<!-- notecardId: 1703778204118 -->

📢 参考答案：

### git 是什么

- Git 是目前最先进的**分布式版本控制系统**，它**用于跟踪和记录项目文件的更改历史**，有了它，你可以提交修改，撤销修改，比较修改、合并修改，甚至恢复到过去的某次提交版本

### 为什么要使用 git

1. **版本控制**
   - git 允许你**查看**过去的版本，**比较**不同版本的差异，甚至可以**恢复**到过去的某个版本
2. **多人协作开发**
   - git 的分布式特性使得**多人**可以同时在**同一个项目**上开发，每个人可以先在本地开发，然后将更改推送到共享的仓库进行合并
3. **分支管理**
   - git 的分支功能使得你可以在**不影响主分支**（通常是 master 或 main 分支）的情况下，**创建新的分支进行功能开发和测试**

### git 的工作区域

- git 共有四个工作区域，三个本地工作区域和一个远程工作区域
  1. **工作区（Workspace）**
     - 指的是你**本地存放代码的目录**，它包含了项目的所有文件
  2. **暂存区（Stage/Index）**
     - 作用是**临时存放你的改动**，事实上它只是一个文件，**存放在隐藏目录 .git 下的 index 文件（.git/index）中**，所以我们把暂存区有时也叫作索引（index），当你执行 git add 命令时，更改会被添加到暂存区，等待被提交
  3. **本地仓库（Local Repository）**
     - 指的是工作区内的**隐藏目录 .git**，这个不算工作区，而是 git 的本地仓库，当你执行 git commit 命令时，暂存区的更改会被**永久记录**到仓库中
  4. **远程仓库（Remote Repository）**
     - 远程仓库是一个代码托管的服务器（比如：Github、Gitee），可**与本地仓库进行数据同步**，当你执行 git push 命令时，本地仓库的更改会推送到远程仓库

## git 常用命令

<!-- notecardId: 1703778204127 -->

📢 参考答案：

- 前端开发中，常用的 Git 命令如下：
  | 操作 | Git 命令 | 描述 |
  | ---------------- | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
  | 克隆仓库 | `git clone <仓库地址>` | 将远程仓库复制到本地 |
  | 初始化新仓库 | `git init` | 在当前目录下创建一个新的 Git 本地仓库 |
  | 查看状态 | `git status` | 显示文件的状态（如未跟踪、已修改、已暂存） |
  | 查看提交历史 | `git log` | 显示所有的提交日志 |
  | 添加文件到暂存区 | `git add <文件>` 或 `git add .` | 将一个或所有文件添加到暂存区 |
  | 提交更改 | `git commit -m "提交信息"` | 将暂存区内容提交到本地仓库 |
  | 推送更改 | `git push <远程仓库名> <分支名>` | 将本地仓库推送到远程仓库的指定分支 |
  | 拉取更改 | `git pull <远程仓库名>` | 拉取远程仓库**当前分支**的最新提交（远程有而本地还没有的）到本地，并**自动合并**到当前分支<br>注：这个命令**不会拉取所有的远程分支**，若没有指定分支名，则默认就是当前分支 |
  | 抓取更改 | `git fetch <远程仓库名>` | 拉取远程仓库**所有分支**的最新提交（远程有而本地还没有的）到本地，但是**不自动合并**到当前分支 |
  | 合并分支 | `git merge <分支名>` | 将不同分支合并到当前分支 |
  | 查看分支 | `git branch` | 查看所有的分支名 |
  | 创建分支 | `git branch <分支名>` | 创建一个新分支 |
  | 切换分支 | `git checkout <分支名>` | 从一个分支切换到另一个分支 |
  | 创建并切换分支 | `git checkout -b <分支名>` | 创建新分支，同时切换到该分支 |
  | 撤销工作区更改 | `git restore <file>` 或 `git restore .` | 撤销一个或所有工作区文件的更改 |
  | 撤销所有更改 | `git revert <某次提交 ID>` | 创建一个新提交来撤销指定提交所做的所有更改<br>注：该命令是通过反向操作来达到撤销 commit 的相同效果，且暂存区和工作区的更改不会保留 |
  | 查看变更 | `git diff` | 比较**工作区**文件和**暂存区**文件的差异 |
  | 暂存更改 | `git stash` | 暂时将所有未提交的更改（包括**暂存的和未暂存**的，**不包括未跟踪**的）存储在 .git 目录内的一个特殊区域 |
  | 应用暂存的更改 | `git stash pop` | 将所存储的暂存内容恢复到工作目录，并从存储区域内删除该暂存内容 |

## 请介绍一下多人协同开发的合并冲突问题

<!-- notecardId: 1703865960077 -->

📢 参考答案：

### 为什么会出现合并冲突

- 在不同分支上，对**同一文件**中的**同一地方**做了**不同修改**（比如：同时修改了同一个文件的同一行），那么在合并分支的时候就会出现冲突，这时就需要手动解决冲突

### 合并分支的分类及其对应的实际场景和解决方法

1. **本地冲突**

   - 定义
     - 指的是**合并两个本地分支时**所产生的冲突
   - 实际场景
     - 开发者 A 和 B 都从 `master` 分支创建了各自的新分支，分别命名为 `feature-a` 和 `feature-b`
     - 他们各自在自己的分支上开发，并**对同一个文件的同一部分进行了修改**
     - A 和 B 完成了他们的功能开发，并将 `feature-a` 和 `feature-b` 分支推送到的远程仓库
     - 此时，项目负责人使用 git fetch 拉取所有最新的分支到本地，并使用 git merge 将 `feature-a` 分支合并到 `master` 分支，当他尝试将 `feature-b` 分支也合并到 `master` 分支时，由于 `master` 分支中的目标文件已经包含了 A 的修改，**Git 无法自动解决两个分支对同一部分代码的不同修改**，因此产生了冲突。
   - 解决方法
     1. 根据 Git 所标记出的冲突部分，通过**选择/编辑**，手动解决冲突的地方
        ![](../Media/git%20%E5%90%88%E5%B9%B6%E5%86%B2%E7%AA%81.png)
     2. 保存文件，并将其**添加到暂存区**
     3. **提交**到本地仓库
     4. **推送**到远程仓库

2. **远程冲突**
   - 定义
     - 指的是**将本地分支推送到远程仓库时**，由于与其他开发者**修改了同一文件的同一处代码**所产生的冲突
   - 实际场景
     - 开发者 A 完成了修改，并将更改推送到了远程仓库的 `master` 分支
     - 开发者 **B 在 A 推送到远程仓库后**，也完成了修改，**且修改了同一文件的同一处代码**，当 B 尝试推送更改到远程仓库的 `master` 分支时，**Git 会拒绝推送**
   - 解决方法
     1. 使用 **git pull** 命令拉取远程仓库最新更改
     2. 根据 Git 所标记出的冲突部分，通过**选择/编辑**，手动解决冲突的地方
        ![](../Media/git%20%E5%90%88%E5%B9%B6%E5%86%B2%E7%AA%81.png)
     3. 保存文件，并将其**添加到暂存区**
     4. **提交**到本地仓库
     5. **推送**到远程仓库

## 请介绍一下 git stash 命令

<!-- notecardId: 1703865960082 -->

📢 参考答案：

### 作用

- 将**工作区**和**暂存区**的**所有更改**（**不包括未跟踪的新增文件**）暂时存储到在 .git 目录内的一个栈上

### 应用场景

1. **切换其他分支**
   - 当你**忘记新建分支，而在主分支 `master` 上开发功能**时，这时你就可以执行 `git stash` **暂存工作区和暂存区的所有更改**，然后切换到新建的分支，再执行 `git stash pop` 取出暂存的内容到新建的分支上
2. **清理工作区**
   - **在拉取新代码或者合并分支前**，一般会先执行 `git stash` 命令来临时清理工作区的修改，这样就可以**排除掉那些由于本地新修改内容所导致的合并冲突**，使得解决合并冲突更省事点
3. **临时切换任务**
   - 比如你正在开发一个新功能，**突然接到一个紧急开发任务**，这时你就可以执行 `git stash` **暂存工作区和暂存区的所有更改**，然后进行紧急任务的开发，完成后，通过 `git stash pop` 取出暂存的内容继续开发

## 请介绍一下抓包

### 抓包的定义

- 指的是通过工具或技术**捕获和分析网络请求和响应数据的过程**

### 抓包的目的

- 查看应用程序与服务器之间请求和响应的内容、状态码、请求时间、响应时间等信息，以便进行**调试和分析**

### 抓包的工具

- PC 端应用
  - 使用 **Chrome 的调试工具**来抓包
- 移动端应用：
  - 使用 **Fiddler**（Windows）、**Charles**（Mac OS）来抓包

### linux 的应用场景

- **当测试机或者线上机出现问题，且本地又无法复现问题时**，就需要去**登录**测试机或者线上机进行**排查**，而**大部分线上机和测试机都是 linux 系统**，所以就需要使用 linux 命令来操作
  > 注：
  >
  > 1. 通常项目代码会**先在测试机上运行**，没有问题后，**再放到线上机上运行**
  > 2. 为了保证运行结果的一致性，**测试机与线上机的系统要尽量保持一致**
