# Front-End-Interview::09-ES6

## var 和 let、const 的区别

<!-- notecardId: 1704382931584 -->

📢 参考答案：

`var`、`let`、`const` 三者的区别主要体现在**变量重复声明、变量提升、变量修改、变量初始化、块级作用域**这几个方面

| 特性         | var                            | let                          | const                        |
| ------------ | ------------------------------ | ---------------------------- | ---------------------------- |
| 变量重复声明 | **允许**在同一作用域内重复声明 | 不允许在同一作用域内重复声明 | 不允许在同一作用域内重复声明 |
| 变量提升     | **会提升**                     | 不会提升                     | 不会提升                     |
| 变量修改     | 可以修改                       | 可以修改                     | **不可以修改**               |
| 变量初始化   | 非必须                         | 非必须                       | **必须**                     |
| 块级作用域   | **无**                         | 有                           | 有                           |

> 注：
>
> 1. `const` 声明的常量不能修改指的是对于值类型不能修改其值，而对于**引用类型**（如对象和数组），**不能修改的是引用地址，其内容是可以修改的**
> 2. 变量初始化指的是**声明的同时赋值**

## 请介绍一下 ES6 的模块化

<!-- notecardId: 1704121798928 -->

📢 参考答案：

### 模块化的定义

- 指的是将**大文件拆分成多个文件**，不同文件之间**相互隔离不影响**，在**需要时**，使用相关命令进行**引入**，形成依赖关系

### 为什么需要模块化

1. 解决不同 js 文件之间的**相同变量名因冲突而覆盖**的问题
2. 实现不同 js 文件之间**互相引用**，进而提高代码**复用性**、**可维护性**

### 模块化的相关规范

- AMD、CMD、CommonJS、ES6
  > 注：
  >
  > 1. AMD 和 CMD 模块化规范适用于浏览器端，**CommonJS** 适用于**服务器端**，而 **ES6** 模块化规范**既适用于浏览器端，也适用于服务器端**
  > 2. 目前 **AMD 和 CMD** 已经落伍，**不推荐使用**

### ES6 模块化的导入导出方式

1. **默认导出与导入**

   - 定义

     - 指的是使用关键字 **export default** 导出，使用 import 导入，且导入语句中的接受名称**不使用花括号包裹**

   - 语法格式

     ```javascript
      // 默认导出
      export default 变量名/函数名
      或者
      export default {
        变量名/函数名,
        ...
      };

      // 默认导入
      import 接受名称 from '所要导入的模块文件地址';
     ```

     > 注：默认导出关键字 export default 在每个 js 文件中**只允许使用一次**；

   - 示例

     ```javascript
     // 模块A：moduleA.js
     const a = 10;
     function b() {
       console.log('Hello World!');
     }
     export default {
       a,
       b,
     };

     // 模块B：moduleB.js
     import myDefaultExport from './moduleA.js';
     console.log(myDefaultExport); // { a: 10, b: [Function: b]}
     ```

2. **按需导出与导入**

   - 定义

     - 指的是使用关键字 **export** 导出，使用 import 导入，且导入语句中的接受名称**使用花括号包裹**

   - 语法格式

     ```javascript
     // 按需导出
     export 代码
     或者
     export {
       变量名/函数名,
       ...
     };

     // 按需导入
     import { 接受名称 1, 接受名称 2, ... } from '所要导入的模块文件地址';
     ```

     > 注：
     >
     > 1. 按需导出**关键字 export** 在每个模块文件中**可以使用多次**
     > 2. 按需导入中的接收名称必须和所导入的模块文件中按需导出的**名称一致**
     > 3. 按需导入中若想**更改接收名称**，通过格式「**接收名称 as 新名称**」来更改
     > 4. 按需导入的 import 语句**不是解构赋值，而是命名导入**，与解构赋值的区别：import 语句中**接收的名称必须是**所导入模块中**定义的变量名，不能是对象的属性名/方法名**，而解构赋值可以直接解构出对象的属性名/方法名

   - 示例

     ```javascript
     // 模块C：moduleC.js
     export const c = 42;
     export const d = 77;
     export const e = 88;

     // 模块D：moduleD.js
     import { c, d as dd, e } from './moduleC.js';
     console.log(c, dd, e); // 42 77 88
     ```

3. **不导出，直接导入并执行代码**

   - 定义

     - 指的是**不使用**关键字 **export 或者 export default** 导出，直接使用 import 导入模块，并执行里面的代码

   - 应用场景

     - 比如某模块是一个**实现某个具体功能模块**，你想直接导入并执行模块内的代码，而**不想通过定义接收名称接收后，再调用来执行**，就可以使用该方法
     - 又或者某模块是一个 **CSS 模块**，你也可以使用方法

   - 语法格式

     ```javascript
     import '所要导入的模块文件地址';
     ```

     > 注：导入的模块文件类型还可以是 **css** 等

   - 示例

     ```javascript
     // 模块E：moduleE.js
     for (let i = 0; i < 3; i++) {
       console.log(i);
     }

     // 模块F：moduleF.js
     import './moduleE.js'; // 输出：1 2 3
     ```

## 数据结构中有序和无序的区别

<!-- notecardId: 1704556462716 -->

🔍 所考察的知识点：数据结构

📢 参考答案：

| 关键词       | 有序                                                     | 无序                                                       |
| ------------ | -------------------------------------------------------- | ---------------------------------------------------------- |
| 定义         | 指的是元素**按照顺序排列**，谁前谁后是有区别的           | 指的是元素**随机放置**，谁前谁后没有区别                   |
| 顺序遍历     | 是                                                       | 否                                                         |
| 操作数据速度 | **查找、插入和删除速度较慢**（因为可能会导致元素的移动） | **查找、插入和删除操作速度较快**（因为不会涉及元素的移动） |
| 常见数据结构 | **数组**、链表、有序集合、树结构（如二叉搜索树）         | **对象**、哈希表、无序集合、堆                             |
| 使用场景     | 当需要**频繁顺序遍历**，而查找、插入和删除操作较少时     | 当需要**频繁查找、插入和删除操作**，而不需要顺序遍历时     |

## Map 和 Object 的区别

<!-- notecardId: 1704556462720 -->

🔍 所考察的知识点：数据结构

📢 参考答案：

| 关键词         | Map                                                                                          | Object                                                                                                      |
| -------------- | -------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| 键的类型       | **任意类型**，可以是函数、对象、数组等                                                       | **只能是字符串**                                                                                            |
| 数据结构       | **有序结构**（但是查找、插入和删除速度**并不慢**）                                           | **无序结构**                                                                                                |
| 操作数据的 API | 新增/修改：map 对象.**set()** <br> 删除：map 对象.**delete()** <br> 查找：map 对象.**get()** | 新增/修改：**object 对象.属性 = 值** <br> 删除：**delete object 对象.属性** <br> 查找：**object 对象.属性** |

## Set 和数组的区别

<!-- notecardId: 1704556462724 -->

🔍 所考察的知识点：数据结构

📢 参考答案：

| 关键词         | Set                                                                                                                                                                                                                 | 数组                                                                                                                                                                                                                                  |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 唯一性         | **不允许**存在重复元素                                                                                                                                                                                              | **允许**存在重复元素                                                                                                                                                                                                                  |
| 数据结构       | **无序结构**                                                                                                                                                                                                        | **有序结构**                                                                                                                                                                                                                          |
| 索引号         | **无**（因为 Set 是无序结构）                                                                                                                                                                                       | **有**                                                                                                                                                                                                                                |
| 操作数据的 API | 新增：**set 对象.add()** <br> 删除：**set 对象.delete()** <br> 修改：无法直接修改，得**先删除，再添加** <br> 查找：无法直接获取，可通过 **set 对象.has()** 判断是否存在，或者通过 **set 对象.forEach()** 遍历来获取 | 新增：**数组.push()、数组.unshift()、数组.splice()** <br> 删除：**数组.pop()、数组.shift()、数组.splice()** <br> 修改：**数组[索引号] = 值** <br> 查找：可以通过**数组[索引号]** 直接访问，也可以通过**数组.includes()** 判断是否存在 |
